# GSW Stats Website Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a dark-themed, premium multi-page website showcasing Golden State Warriors players (current + legends) with career stats, using vanilla HTML/CSS/JS.

**Architecture:** Static multi-page site with shared CSS and JS. Player data lives in a single `data.js` file. Each HTML page loads the shared data and `app.js` which handles player selection, rendering, and comparison logic. No build tools, no frameworks. All data is hardcoded — no user input is rendered, so DOM string building from trusted data is safe.

**Tech Stack:** HTML5, CSS3, vanilla JavaScript (ES6+)

---

## File Structure

```
gsw-stats/
  index.html          — landing page (hero, featured player, team facts)
  roster.html         — current roster (sidebar + detail panel)
  legends.html        — legends (sidebar + detail panel)
  compare.html        — player comparison (dropdowns + bar charts)
  css/
    style.css         — all styles
  js/
    data.js           — player data object (hardcoded, trusted)
    app.js            — all page logic (player selection, rendering, comparison)
  img/                — placeholder directory
```
